## Как запустить

1. Запустите игру.
2. На игровом экране появится картинка, разбитая на пустые слоты, и набор кусочков пазла снизу.
3. Зажми кусочек пазла и перетащи его к нужному месту на картинке.
4. Если место правильное, кусочек автоматически встанет в слот и зафиксируется.
5. Если место неправильное, кусочек вернётся обратно в нижнюю область.
6. Если нужна помощь, нажми кнопку "Hint" игра подсветит нужный слот и соответствующий кусочек.
7. Когда все кусочки будут на местах, уровень завершится.
8. После завершения можно нажать кнопку Далее, чтобы перейти к следующему уровню, или открыть выбор уровней.


Дополнительные настройки не нужны. 

Что реализовано?

Обязательные элементы

1. 2D-пазл на Canvas UI.
2. Drag & drop кусочков пазла.
3. Snap кусочка в правильный слот.
4. Возврат кусочка назад, если место неправильное.
5. Несколько уровней с разными изображениями.
6. Состояние завершения уровня.
7. Переход к следующему уровню.
8. Часть арта сгенерирована нейросетью.

Дополнительно реализовано

1. Экран выбора уровня.
2. Индикатор прогресса сборки.
3. Подсказка: подсвечивается нужный слот и соответствующий кусочек.
4. Звук на нажатие кнопки.
5. Звук при правильной установке кусочка.
6. Звук при завершении уровня.
7. VFX салюта при завершении уровня.
8. Адаптация под портретный мобильный формат.
9. Разные размеры пазла под разную сложность через `rows` и `columns` в настройках уровня.

Использование ИИ

В процессе разработки я использовал ИИ для ускорения работы над кодом, структурой проекта и арт-частью.

Для кода использовал ChatGPT / Codex:
- помог составить архитектуру простого Unity-прототипа;
- написал и доработал скрипты для drag & drop, snap, уровней, подсказки, выбора уровня и VFX;
- помог исправить ошибки в процессе, например `NullReferenceException` при инициализации кусочков и проблемы с размером/раскладкой UI.

Для арта использовал генерацию изображений нейросетью:
- были сгенерированы детские квадратные иллюстрации для уровней;
- затем изображения использовались как основа пазлов и нарезались на части внутри Unity.

Примеры промптов для помощи с кодом

1. Сделай простой Unity 2D пазл на Canvas UI: кусочки можно перетаскивать, при попадании в правильный слот они снапаются, при ошибке возвращаются назад.
2. Добавь экран выбора уровня, чтобы кнопки уровней создавались автоматически из массива ScriptableObject уровней.
3. Надо написать Unity C# скрипт для UI-кусочка пазла. Кусочек должен работать через EventSystem и интерфейсы IBeginDragHandler, IDragHandler, IEndDragHandler. 
При drag он должен двигаться по Canvas с учетом canvas.scaleFactor, при отпускании вызывать PuzzleManager.TryPlacePiece(this). 
Код должен быть простым, читаемым, без лишних абстракций.
4. Проверь возможные причины NullReferenceException в методе PuzzlePiece.Init. 
Скрипт работает с Image, CanvasGroup, RectTransform и получает sprite из PuzzleManager.

Как тестировал?

Я проверял основные игровые сценарии:
запуск игры и загрузка первого уровня;
открытие экрана выбора уровня;
выбор разных уровней;
перетаскивание кусочков по экрану;
правильный snap кусочка в нужный слот;
возврат кусочка, если отпустить его не в том месте;
работа подсказки;
обновление прогресса;
завершение уровня;
переход на следующий уровень;
повторное прохождение после смены уровня;
воспроизведение звуков;
запуск VFX салюта после сборки.

В процессе тестирования были найдены и исправлены проблемы
NullReferenceException у кусочка пазла, когда Image не был назначен в prefab;
кусочки могли отображаться слишком маленькими или неудобно располагаться;
подсказка сначала подсвечивала только слот, затем была доработана, чтобы показывать и нужный кусочек;
салют сначала появлялся почти в одной точке, после чего был доработан для появления в разных частях экрана;
экран выбора уровня был вынесен в отдельную панель, чтобы его можно было удобно открыть и закрыть.
Время разработки
На прототип ушло примерно: 2 часа.
